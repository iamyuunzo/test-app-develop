## 1. CRUD 흐름 쪼개기

### 1️⃣ Create — “질문 생성”

**의미**

* 사용자가 질문 1개를 입력함
* 이 질문이 하나의 “세션”이 됨

**앱에서는**

* 질문 입력 화면
* 저장 버튼

👉 이 순간:

> “AI 비교 세션이 생성됨”

---

### 2️⃣ Read — “4개의 AI 답변을 본다”

**의미**

* 하나의 질문에 대해
* 4개의 AI 응답이 있음

**현실 구현**

* 진짜 AI 응답 ❌
* **더미 데이터 4개** ⭕

예:

* GPT 응답 (mock)
* Claude 응답 (mock)
* Gemini 응답 (mock)
* LLaMA 응답 (mock)

👉 이걸 **세로 스크롤 / 카드 4개**로 보여줌

이 시점에서 멘토는 이렇게 봄:

> “아, 이 사람은
> ‘하나의 질문 → 여러 AI 응답’ 구조를 명확히 이해했네”

---

### 3️⃣ Update — “대화 횟수 증가”

**의미**

* 질문을 추가로 던질 때마다
* ‘남은 횟수’가 줄어듦

**앱에서는**

* “남은 질문 횟수: 2 / 5”
* 버튼 누를 때마다 감소

실제 AI 대화 ❌
→ **카운트 상태만 변경** ⭕

👉 여기서 중요한 건:

> **‘제한이 있는 UX’가 존재한다는 점**

---

### 4️⃣ Update — “AI 하나 선택”

**의미**

* 제한 도달
* 더 이상 비교 불가
* **선택 강제**

**앱에서는**

* “이 AI를 선택” 버튼
* 하나 선택하면 나머지 비활성화

👉 이게 AIQ의 **가장 중요한 분기점**

멘토 시점:

> “오, 서비스 정책을 UX로 풀었네?”

---

### 5️⃣ Read — “요약된 프롬프트 확인”

**의미**

* 지금까지의 대화가
* 하나의 요약된 프롬프트로 정리됨

**현실 구현**

* 진짜 요약 ❌
* **미리 만들어둔 요약 텍스트** ⭕

예:

> “위 대화를 요약해,
> React Native 상태관리 기준을 안드로이드 기준으로 정리해줘”

👉 여기서 중요한 건 “결과물이 있다”는 것

---

### 6️⃣ Delete (선택) — “세션 종료”

**의미**

* 비교 세션 삭제
* 다시 처음으로

**앱에서는**

* “세션 종료” 버튼

---

## 2. CRUD 최종 요약 정리

| 단계    | CRUD   | 의미          |
| ----- | ------ | ----------- |
| 질문 입력 | Create | AI 비교 세션 생성 |
| 응답 보기 | Read   | 4개 AI 응답 조회 |
| 질문 횟수 | Update | 상태 변화       |
| AI 선택 | Update | 정책 반영       |
| 요약 확인 | Read   | 결과 소비       |
| 종료    | Delete | 세션 제거       |

---

## 3. 전체 화면 구성 한눈에 보기

```
[ Screen 1 ]
질문 입력
   ↓
[ Screen 2 ]
4개 AI 응답 비교
   ↓
[ Screen 3 ]
AI 선택 + 요약 프롬프트
   ↓
[ (선택) Screen 4 ]
세션 종료 / 초기화
```

---

## 4. 🟦 Screen 1 — 질문 입력 (Create)

### 목적

* AI 비교 세션 생성
* “질문 1개 → 여러 AI”의 시작점

### 화면 요소

* 질문 입력 TextArea
* “질문 던지기” 버튼

### 상태 관점 (중요)

* `question` 생성됨
* `session` 시작됨
* `remainingCount = 5`

---

## 5. 🟩 Screen 2 — 4개 AI 응답 비교 (Read + Update)

### 목적

* AIQ의 **정체성 화면**
* 4개 답변을 한 번에 비교

---

### 화면 레이아웃 (둘 중 하나 선택)

#### 옵션 A (추천 ⭐)

* 세로 스크롤
* 카드 4개 연속

```
[ GPT 응답 카드 ]
[ Claude 응답 카드 ]
[ Gemini 응답 카드 ]
[ LLaMA 응답 카드 ]
```

#### 옵션 B

* 2x2 그리드
* 한 화면에 4개

---

### 화면 요소

* AI 이름
* 더미 응답 텍스트
* 남은 질문 횟수 표시

  ```
  남은 질문: 3 / 5
  ```
* “추가 질문” 버튼 (눌러도 mock)

---

### 상태 변화

* 질문 추가 → `remainingCount--`
* 실제 AI 호출 ❌
* **카운트만 변경 ⭕**

---

## 6. 🟨 Screen 3 — AI 선택 + 요약 프롬프트 (Update + Read)

### 목적

* 비교 종료
* **강제 선택**
* 결과물 제공

---

### 화면 요소

* “질문 횟수 초과” 안내 문구
* 각 AI 카드에

  * “이 AI 선택” 버튼

👉 하나 선택 시:

* 나머지 비활성화
* 선택 상태 표시

---

### 요약 프롬프트 영역

* 미리 만들어둔 요약 텍스트 표시

예:

> “위 대화를 요약해
> React Native 상태관리 기준을 안드로이드 중심으로 정리해줘”

---

### 버튼

* “프롬프트 복사”
* “외부 AI로 이동” (딥링크는 주석으로 대체 가능)

---

## 7. ⬛ Screen 4 (선택) — 세션 종료 / 초기화 (Delete)

### 목적

* 세션 정리
* 처음으로 돌아가기

### 화면 요소

* “새 질문 시작” 버튼
